# 设计文档：首页 Hero Col2 整体旋转动效 (方案 A)

> 日期：2026-05-30
> 目标：实现当用户向下滚动时，`hpc_col2` 整体向左旋转并最终呈水平方向展示的 GSAP 动效。

## 1. 核心需求
- **触发条件**：用户在首页进行垂直滚动。
- **视觉逻辑**：`hpc_col2` 及其子元素（手机、平板、背景色块）作为一个整体，从初始的倾斜角度（30°）转正到水平角度（0°）。
- **空间补偿**：随着旋转，整体向右侧平移，以腾出左侧空间，增强“展开”的视觉冲击力。

## 2. 初始状态 (Initial State)
- **背景旋转**：`--hero-col2-bg-rotation: 30deg` (定义于 `globals.css`)。
- **定位**：由现有的弹性布局和 CSS 变量控制的桌面端布局。

## 3. 动画规格 (GSAP + ScrollTrigger)

### 3.1 触发配置
- **Trigger**: `.calypso-block--EW19-tile-1` (Hero 容器)。
- **Start**: `top top` (容器触顶开始)。
- **End**: `+=420` (持续滚动 420px 完成动画)。
- **Pin**: `true` (动画期间固定 Hero 区域)。
- **Scrub**: `0.8` (顺滑跟随滚动条)。

### 3.2 整体变换 (#hpc_col2)
- **旋转 (Rotate)**: `30deg` -> `0deg` (向左/逆时针转正)。
- **位移 (X)**: `0` -> `+180px` (向右平移，补偿旋转后的重心偏移)。
- **缩放 (Scale)**: `1` -> `0.96` (轻微缩小，增加纵深感)。
- **支点 (Origin)**: `center center`。

### 3.3 分层视差 (内部微动)
在整体旋转的同时，内部层级进行差异化 X 轴位移：
- **手机层 (#hpc_col2 .hpc-phone)**: `+112px` (最快)。
- **背景层 (#hpc_col2 .hpc-pics--no-rotated)**: `+96px` (中等)。
- **平板层 (#hpc_col2 .hpc-tablet)**: `+72px` (最慢)。

## 4. 响应式与降级
- **断点**: 仅在宽度 `>= 992px` (桌面端) 时启用。
- **辅助功能**: 若用户开启 `prefers-reduced-motion: reduce`，则取消旋转效果，仅保留微小的位移。
- **性能**: 对 `#hpc_col2` 及其所有运动子层强制应用 `will-change: transform`。

## 5. 验收标准
1. 滚动时，背景黄色块能平滑地转到水平。
2. 手机和平板随之转正，且能感觉到它们相对于背景有轻微的横向滑动。
3. 滚动停止时，动画立即停在当前进度，无跳动。
4. 重新刷新页面或缩放窗口，ScrollTrigger 能正确重置位置。
